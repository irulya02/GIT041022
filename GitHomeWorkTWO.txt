
1. Чтобы отказаться от изменений в файле или файлах используем команду 
git checkout + название файла - отмена изменений в конкретном файле
 или 
git checkout .  - отмена изменений во всех файлах

2. Чтобы вернуть файл из индекса в рабочий каталог используем команду 
git restore  --staged  + название файла
или
git reset 

3. Чтобы отменить ошибочный коммит используем команду 
git reset HEAD^ - ставим галочек (керит) столько на сколько шагов назад нам нужно вернуться
git reset HEAD~2 -  ~ + количество шагов назад, выраженное цифрой


4. Чтобы создать новую ветку используем команду 
git branch имя ветки
или git checkout -b имя ветки

5. Отличия в том что в nonfast-forward merge имеется один результирующий коммит, который объединяет две ветки  

6. Конфликт слияния веток может вознинкуть тогда, когда мы делали изменения в файле на одной ветке, и затем выполнили некоторые изменения в этом же файле на другой ветке, в таком случае git не понимает что ему необходимо сохранить. Мы делаем изменения в файле в ручную, например удаляем все то что добавили в разных ветках и затем выполняем команду git add и git commit -m  

7. Удалить новую ветку можно с помощью команды git branch -d имя ветки
